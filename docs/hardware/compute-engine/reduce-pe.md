# FPE 浮点运算引擎

## 1. 术语说明

| 术语 | 说明 |
|------|------|
| FPE | Floating-Point Engine，浮点运算引擎，CUTE 中每个 PE 的核心计算逻辑 |
| FReducePE | FPE 的源码实现类名 |
| 浮点路径 | 标准浮点计算路径（FP16/BF16/TF32），经过 26-bit 对齐和归约 |
| 整数路径 | INT8 计算路径（I8×I8 / I8×U8 / U8×I8 / U8×U8），走整数乘积分支与归约 |
| ReduceWidth | 归约通道位宽（默认 512 bit） |
| CmpTree | 指数比较树，位并行实现，用于浮点对阶时查找最大指数 |
| RawFloat | FPE 内部统一的浮点表示格式，浮点类型解码后统一为此格式 |
| ULP | Unit in the Last Place，最小精度单位 |


## 2. 设计概述

FPE 是 CUTE 中每个 PE 的核心计算逻辑，执行向量内积并累加到 C 矩阵元素上：

```
D[m][n] = Σ_k(A[m][k] × B[k][n]) + C[m][n]
```

### 四阶段逻辑架构

FPE 的执行过程划分为四个核心逻辑阶段。各阶段之间、以及阶段内部的长延迟逻辑（如大位宽加法器、CmpTree）处可灵活插入寄存器，以适配不同的时序目标：

![FPE图](fpe.png)

### 双流设计目标

| 设计维度 | 浮点路径 | 整数路径 |
|---------|--------|--------|
| 目标格式 | FP16 / BF16 / TF32 | I8×I8 / I8×U8 / U8×I8 / U8×U8 |
| 对阶方式 | CmpTree + 26-bit 对齐移位 | 无对阶 |
| 吞吐量 | 1X（基准） | 1X（基准） |
| Scale 支持 | 否 | 否 |
| 硬件复用 | 共享对齐寄存器、归约树、乘法器 | 复用浮点路径的乘法器和加法器 |

## 3. FPE 参数

FPE 的关键参数定义：

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `ReduceWidth` | 512 | 归约输入位宽 |
| `ResultWidth` | 32 | 归约输出位宽 |
| `cmptreelayers` | 4 | CmpTree 在前阶段的比较层数 |
| `P3AddNum` | 4 | 最终归约级加法器数量（固定值） |

**派生参数：**

| 参数 | 计算公式 | 含义 |
|------|---------|------|
| `P3AddNum` | `4` | 最终归约级加法器数量 |
| `P2AddNum` | `ReduceWidth / (P3AddNum × 16)` | 部分归约每组元素数 |

## 4. 阶段 A：预处理与解码

阶段 A 负责将不同格式的输入数据统一解码为内部表示，并行完成尾数乘积和指数预算。

### 4.1 多格式解码

根据 `opcode` 将输入向量（ReduceWidth 位宽）并行解码。浮点数据解码为 `RawFloat`，INT8 数据解码为独立的 `Int8Vec`，后续按路径归约。

`RawFloat` 内部表示：

```
RawFloat(expWidth=8, precision=11) {
    sign        : 1 bit         // 符号位
    exp         : SInt(8.W)     // 真实指数（已减去 bias，有符号）
    signed_sig  : SInt(12.W)    // 含隐含 1 的有符号尾数
    exception {
        is_nan  : Bool          // 指数全 1 且尾数非零
        is_inf  : Bool          // 指数全 1 且尾数全零
        is_zero : Bool          // 所有位为零
    }
}
```

**解码方式：**

| 数据类型 | 解码方法 | 关键处理 |
|---------|---------|---------|
| FP16 (E5M10) | 提取 S/E/M，5-bit 指数扩展到 8-bit | 指数真实值不变（bias 自动换算） |
| BF16 (E8M7) | 提取 S/E/M，7-bit 尾数补 3 个零 | 尾数零扩展至 10-bit，保持数值 |
| TF32 (E8M10) | 取 32-bit 字的高 19 bit | 直接使用 |
| INT8 | 按 opcode 进行有符号/无符号扩展，写入 `Int8Vec(9-bit)` | 进入整数归约路径 |

**解码的核心逻辑** `RawFloat.fromUInt`：

- 输入原始 bit 流，参数化 `expWidth` 和 `precision`
- 指数域全零时强制设为 1 后再减 bias，使零和 denormal 的指数不会过小导致后续对阶移位爆炸
- 尾数拼接隐含 1：`Cat(0, nz, mantissa)`，其中 `nz` 表示指数是否非零（即是否为正常数）
- 同步检测 NaN、Inf、Zero 三种异常

### 4.2 双路径并行

**浮点路径：** FP16/BF16/TF32 的主路径，解码后的 `RawFloat` 送入后续的 26-bit 对齐和归约流程。

**整数路径：** I8/U8 数据经 `Int8Vec` 进入整数乘积分支，和浮点路径在后级归约树中复用加法资源。

### 4.3 尾数乘积运算

解码后，对 A 和 B 的尾数执行逐元素乘法。乘法器位宽按最宽格式的需求设计（12 bit × 12 bit），窄格式数据自动通过零扩展适配，无需额外控制。

乘积结果分为两组：浮点分支 `Product0(SInt(23.W))`，整数分支 `Product1(SInt(17.W))`，并在后续阶段完成归约。

### 4.4 指数预算

同步计算每对乘积的有效指数：

```
有效乘积指数 = A.exp + B.exp
```

指数以**偏移无符号**表示（`真实值 + 255`），使负指数也能用位运算高效比较。零值的乘积指数强制设为 0，避免干扰最大值查找。

### 4.5 异常传播

阶段 A 同步计算每对乘积的异常标志，用于最终结果判定：

| 异常 | 判定条件 |
|------|---------|
| NaN | A_nan ∨ B_nan ∨ (A_inf ∧ B_zero) ∨ (B_inf ∧ A_zero) |
| Zero | (A_zero ∧ ¬B_inf) ∨ (B_zero ∧ ¬A_inf) |
| Inf | (¬A_zero ∧ B_inf) ∨ (¬B_zero ∧ A_inf) |

## 5. 阶段 B：全局对阶与对齐

阶段 B 是浮点运算精度的关键保障，通过比较树和动态移位阵列，将所有乘积对齐到统一的指数基准。

### 5.1 指数收敛（CmpTree）

**目标：** 在全部乘积指数 + C 值指数中找到最大值 `MaxExp`。

**位并行比较策略**：

CmpTree 不使用传统的比较器树，而是采用**位运算掩码**方法：

1. 将所有 9-bit 指数按位排列为 9 组位向量
2. 从最高位（bit 8）开始，逐位生成掩码 `mask`：
   - `mask(0)` = bit8 的非零掩码（保留 bit8=1 的元素位置）
   - `mask(i)` = `mask(i-1) & bit(i)`（取交集，逐步缩小）
3. 若某一层的交集为空，则保留上一层的掩码不变
4. 最终输出 9-bit 编码，每位表示该位在所有有效指数中是否全为 1

**优势：** 全部用位与/或运算实现，不依赖比较器的级联延迟，适合 ASIC 高频实现。

**两级拆分：** 9 层串行掩码可能超出时序预算，拆为两部分跨两个阶段执行：

- **前阶段（cmptreelayers 层）：** 计算前 4 层掩码，输出中间结果
- **后阶段（9 - cmptreelayers 层）：** 接续完成剩余层，输出完整 `MaxExp`

两级之间可插入寄存器，也可根据时序合并为单级执行。

### 5.2 动态偏移计算

```
RightShift[i] = MaxExp - MulExpVec[i]
```

每个乘积需要右移 `RightShift[i]` 位，使尾数对齐到公共指数 `MaxExp`。右移量范围为 0 ~ 511（9-bit）。

### 5.3 26-bit 高精度对齐阵列

**结构：** 使用 33 个 26-bit 寄存器装载对齐后的数据（32 路乘积 + 1 路 C 值）。

**26-bit 位宽构成：**

```
|←—— 23 bit ——→|←3→|
   乘积尾数     低位补零
```

| 位段 | 位宽 | 作用 |
|------|------|------|
| 乘积尾数扩展 | 26 bit | `Cat(Product0(22,0), 0.U(3.W))` 后参与对齐右移 |
| C 尾数对齐 | 26 bit | `CMantissa(25,0).asSInt` 参与对齐右移 |

**精度对齐目标：** 26-bit 对齐宽度与当前实现保持一致，浮点路径统一以该位宽进入后续归约。

**对齐过程：**

1. 乘积尾数扩展为 26-bit：`Cat(Product0(22,0), 0.U(3.W))`
2. C 尾数截取为 26-bit：`CMantissa(25,0).asSInt`
3. 根据 `RightShift` 执行算术右移，得到 33 路对齐值
4. 浮点 opcode 写入对齐结果；整数 opcode 下 `Product0(0..31)` 直通 `pad(26)`，`Product0(32)` 置 0

## 6. 阶段 C：归约与累加

阶段 C 将浮点路径的 33 路 26-bit 对齐数据通过多级加法树归约为单一求和结果，并并行归约整数路径后叠加 C 值。

### 6.1 符号注入

在归约求和之前，根据每路数据的符号位将 26-bit 尾数转换为补码形式：

```
aligned_signed = Mux(sign, -aligned_unsigned, aligned_unsigned)
```

这样后续的加法树只需处理绝对值相加，最终结果的符号在阶段 D 通过最高位恢复。整数路径的 `Product1` 为有符号乘积，直接按补码累加。

### 6.2 多级归约树

**浮点路径：**

33 路 26-bit 数据（32 路乘积 + 1 路 C 值）送入加法树，逐级压缩为单一结果：

```
33 路 26-bit
├── 第 1 级：每组若干元素求和 → 部分和（位宽适当扩展以容纳进位）
├── 第 2 级：部分和继续求和 → 更少的部分和
└── 最终级：全部归约为单一 32-bit 累加结果
```

归约过程中每一级的位宽都会适当扩展，以承载多路数据累加产生的进位，防止溢出。

**整数路径：**

整数路径将 32 路 `Product1(17-bit)` 分组归约为 `ReduceRes1`，并在最终级按 `ReduceResI = ReduceResF + sum(ReduceRes1)` 与浮点主归约结果合并。

| 路径 | 每周期处理元素数 | 对阶 | 归约方式 |
|------|----------------|------|---------|
| 浮点路径 | 基准 | CmpTree + 26-bit 移位 | 浮点加法树 |
| 整数路径 | 基准 | 无 | 整数乘积分支加法树 |

### 6.3 C 值合并

C 值与乘积归约结果统一累加。在浮点模式下，C 以 26-bit 对齐后的尾数参与归约；在最终输出为 INT32 时，C 以原始 32-bit 整数直接相加。

## 7. 阶段 D：后处理与规格化

阶段 D 将归约结果转换为标准 FP32 或 INT32 格式输出。

### 7.1 规格化调整

1. **提取符号**：`ResultSign = 累加结果[31]`
2. **取绝对值**：`UnsignedSig = |累加结果|`
3. **前导零检测（CLZ）**：计算 `UnsignedSig` 中前导零个数
4. **指数调整**：`FinalExp = (MaxExp - 255) - LeadingZeros + 8`，将最高有效位对齐到规格化位置
5. **尾数移位**：根据 CLZ 结果左移或右移尾数，使隐含 1 对齐到正确位置

### 7.2 舍入与异常处理

**舍入模式：** 当前实现采用移位/截位组合，未实现显式 RNE 与 Guard/Round/Sticky 舍入判定。

**异常结果判定：**

| 优先级 | 条件 | 输出值 | 说明 |
|--------|------|--------|------|
| 1（最高） | 存在 NaN，或同时存在 +Inf 和 -Inf | `0x7FC00000` | Quiet NaN |
| 2 | 存在 +Inf（且无 NaN 或 +Inf/-Inf 共存） | `0x7F800000` | 正无穷 |
| 3 | 存在 -Inf（且无更高优先级异常） | `0xFF800000` | 负无穷 |
| 4（最低） | 所有元素均为负零 | `0x00000000` | 当前实现输出 +0 |
| — | 无异常 | 正常计算结果 | — |

**溢出/下溢处理：**

| 条件 | 处理 |
|------|------|
| 指数 > 127 | 上溢为 +Inf / -Inf（按符号） |
| 指数 < -126 | 下溢为 0 或 denormal |
| 指数 ≤ -149 | 完全下溢为 0 |

### 7.3 结果输出

```
io.out = Mux(整数opcode, INT32 累加结果, Mux(有异常, 异常编码, FP32 规格化结果))
```

- **整数模式**（INT8 等）：直接输出 32-bit 累加结果，跳过所有浮点归一化和异常处理
- **浮点模式**：输出 FP32，遇异常时输出对应编码

## 8. 设计亮点

### 8.1 灵活的流水线切分

阶段 A~D 是逻辑划分，不是严格的时钟周期划分。由于 CmpTree（指数比较树）和加法树（Reduction Tree）逻辑较深，设计中预留了"可插拔"的寄存器插入点：

- **宽松时序**：可将多个阶段合并到同一时钟周期
- **紧凑时序**：在阶段内部进一步拆分（如 CmpTree 的两级拆分、归约树的多级切分）

这种灵活性使 FPE 能够适配不同的工艺库和频率目标。

### 8.2 统一的内部数据结构

通过定义一致的 `RawFloat` 内部格式，浮点类型在阶段 A 解码后使用相同的数据路径；INT8 使用独立的 `Int8Vec` 路径。这使得：

- 浮点路径和整数路径在归约阶段可以复用大量的加法器资源
- 不同数据类型之间切换时无需额外的数据通路选择逻辑
- 简化了验证和测试的复杂度

### 8.3 精度对标与验证

当前实现的精度行为以本仓库源码与测试金模型为准。文档中的对齐位宽、移位/截位策略与异常优先级均按 `FReducePE.scala` 的现实现描述。

## 9. 硬件复用策略

### 9.1 乘法器复用

12×12 bit 的浮点尾数乘法器与整数乘积分支共享主流水乘加资源，按 `opcode` 选择具体数据路径。

### 9.2 寄存器复用

33 个 26-bit 对齐寄存器在两种路径下承担不同角色：

| 模式 | 寄存器用途 |
|------|-----------|
| 浮点路径 | 存储移位对齐后的浮点尾数 |
| 整数路径 | 复用同一寄存结构存放扩展后的整数乘积分支数据 |

### 9.3 加法器树复用

归约加法树是浮点路径和整数路径共享的核心资源：
- 浮点路径：处理 26-bit 浮点尾数的符号翻转和累加
- 整数路径：处理整数乘积分支的直接累加

## 10. 数据类型支持

| 数据类型 | 编码 | 输入位宽 | 累加精度 | 输出 | 路径 |
|---------|------|---------|---------|------|------|
| I8 × I8 → I32 | 0 | 8 bit | INT32 | INT32 | 整数路径 |
| FP16 × FP16 → F32 | 1 | 16 bit | FP32 | FP32 | 浮点路径 |
| BF16 × BF16 → F32 | 2 | 16 bit | FP32 | FP32 | 浮点路径 |
| TF32 × TF32 → F32 | 3 | 19 bit | FP32 | FP32 | 浮点路径 |
| I8 × U8 → I32 | 4 | 8 bit | INT32 | INT32 | 整数路径 |
| U8 × I8 → I32 | 5 | 8 bit | INT32 | INT32 | 整数路径 |
| U8 × U8 → I32 | 6 | 8 bit | INT32 | INT32 | 整数路径 |


## 11. 流水线控制

### 11.1 反压机制

FPE 采用寄存器反压机制控制数据流动。各级之间通过 `AllowIn` 信号传播反压：当后级阻塞时，前级暂停接受新数据，形成全流水线的反压链。

输入/输出接口采用 Decoupled 握手协议：

| 接口 | 方向 | 位宽 | 说明 |
|------|------|------|------|
| `AVector` | Input (Flipped Decoupled) | `ReduceWidth` | A 向量数据 |
| `BVector` | Input (Flipped Decoupled) | `ReduceWidth` | B 向量数据 |
| `CAdd` | Input (Flipped Decoupled) | 32 | C 累加值 |
| `DResult` | Output (Decoupled) | 32 | 归约结果 |
| `opcode` | Input | 3 | 数据类型 / 计算模式选择 |
